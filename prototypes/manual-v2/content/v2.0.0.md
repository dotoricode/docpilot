# DocPilot v2.0.0

> 공개일: 2026-07-15 · 설치 파일과 태그는 GitHub Release `v2.0.0`에서 제공합니다.

DocPilot 2.0은 문서를 찾고, 읽고, 편집하고, 변경사항을 검토하는 흐름을 하나의 작업공간으로 다시 구성했습니다. 특정 AI 실행 화면에 맞추는 대신 프로젝트 파일과 사용자의 기본 셸을 중심에 둡니다.

## 1. 문서 중심 워크벤치

프로젝트 탐색기, 열린 문서 탭, 본문 캔버스, 검토 레일과 상태 정보를 한 화면에서 확인할 수 있습니다. 홈에서는 최근 문서를 다시 열거나 빠른 열기로 프로젝트 파일을 찾을 수 있습니다.

**사용 방법**

1. 프로젝트 폴더를 엽니다.
2. 파일 트리에서 문서를 선택하거나 `⌘P`로 빠른 열기를 실행합니다.
3. 상단 탭을 사용해 열어 둔 문서 사이를 이동합니다.

**지원 형식**

- Markdown: Source, Rich, Preview, Outline
- AsciiDoc: Source, Preview, Outline
- JSON: Source, Tree, Format, Validate
- JavaScript, TypeScript, YAML, 일반 텍스트: Source

Rich 모드에서 원문을 안전하게 왕복할 수 없는 Markdown 구문이나 큰 문서는 Source 모드로 엽니다. 지원하지 않는 문법을 임의로 변환하지 않습니다.

## 2. 탭과 Pane 분할

문서 탭을 작업 영역 가장자리로 끌면 놓기 전에 분할 위치를 확인할 수 있습니다. 왼쪽·오른쪽·위·아래 배치를 지원하며, 분할 비율과 Pane 위치는 다시 열었을 때 복원됩니다.

1. 이동할 문서 탭을 잡습니다.
2. 작업 영역 가장자리의 넓은 드롭 영역으로 이동합니다.
3. 파란 미리보기 영역을 확인한 뒤 탭을 놓습니다.
4. 분할선을 끌어 각 문서의 폭이나 높이를 조절합니다.

## 3. 사용자의 기본 셸을 여는 터미널

터미널 Pane은 Codex나 Claude 전용 화면이 아닙니다. 프로젝트 디렉터리에서 사용자의 기본 로그인 셸을 열며, 필요한 CLI를 직접 실행할 수 있습니다.

- 닫힌 터미널은 작업 영역의 `Terminal` 버튼으로 다시 엽니다.
- `+`로 세션을 추가하고 탭으로 세션을 전환합니다.
- 터미널 Pane도 문서 주변의 상하좌우로 이동할 수 있습니다.
- 세션 출력은 순서가 보존되며, 실행 중인 bridge에 렌더러가 다시 연결될 때 제한된 화면 스냅샷을 사용합니다.

전체 DocPilot 재시작 뒤 셸 프로세스 생존은 현재 보장하지 않습니다.

## 4. Source와 Preview에서 변경사항 검토

Diff를 켜면 Source에서는 변경된 줄을, Preview에서는 렌더링된 문서 안의 추가·삭제 부분을 확인할 수 있습니다. 우측 Changes 레일은 변경 지점과 요약을 제공하고 긴 문서의 검토 위치로 이동하는 데 사용합니다.

1. 변경된 문서를 엽니다.
2. Source 또는 Preview를 선택합니다.
3. `Diff`를 켭니다.
4. 필요하면 좌우 비교를 켜 원본과 변경본을 나란히 확인합니다.
5. Changes 레일의 항목을 선택해 해당 위치로 이동합니다.

## 5. 프로젝트 검색

`⌘⇧F`는 프로젝트 파일 이름과 본문을 검색합니다. 포함·제외 glob, 대/소문자, 단어 단위와 정규식 조건을 조합하고 결과에서 문서를 열 수 있습니다.

Preview에서 드래그한 선택은 참고 칩과 근접 복사 메뉴가 나타난 뒤에도 유지됩니다. `Cmd/Ctrl+C`로 선택한 범위만 복사하면 선택 주변에서 완료 피드백을 확인할 수 있습니다.

테마는 라이트와 다크를 모두 지원하며 마지막 선택을 다음 실행과 파일 열기 화면에 적용합니다.

## 업그레이드 후 확인할 항목

- 프로젝트와 최근 문서가 홈에 표시되는지 확인합니다.
- 문서 탭과 터미널 Pane의 기존 배치를 확인합니다.
- Markdown Rich 모드에서 지원하지 않는 구문은 Source 모드로 유지되는지 확인합니다.
- 라이트·다크 테마에서 Preview, Diff, 터미널의 대비를 확인합니다.
