# DocPilot 데모 제작 가이드

## 원칙

- 실제 DocPilot Electron 앱과 임시 Git 프로젝트를 사용한다.
- 시작·완료 상태를 한 영상 안에서 보여 주고, 한 영상에는 한 작업만 담는다.
- 커서 이동은 Playwright로 재현해 같은 영상을 다시 만들 수 있게 한다.
- 1440×900 원본을 보존하고 문서용 포맷을 별도로 만든다.
- 카메라는 `overview`와 `focus` 두 상태만 사용한다. 작은 입력 대상을 읽어야 할 때만 확대하고, Pane·창·앱 사이의 공간 관계가 결과라면 전체 화면으로 돌아온다.
- 한 클립에서 카메라 이동은 보통 두 번 이하로 제한한다. 확대·축소는 250–450ms의 ease-in-out으로 처리하고 장식 목적의 이동은 넣지 않는다.
- 최종 길이를 하나로 맞추지 않는다. 빠른 열기·키 입력은 빠르되 추적 가능하게, 드래그·전환은 자연스럽게, 읽을 결과는 1.2–1.8초, 완료 상태는 약 1.5초 유지한다.
- 커서 이동, 메뉴 탐색, 드래그, 레이아웃 전환은 실제 속도를 유지한다. 로딩과 입력 없는 대기만 최대 2.5배까지 줄일 수 있다.
- 커서가 보이는 동안 점프 컷을 사용하지 않는다.
- `prefers-reduced-motion` 환경에서는 자동 재생하지 않고 포스터를 표시한다.

## 재현 절차

```sh
node scripts/capture-manual-v2-demos.js all
node scripts/process-manual-v2-demos.mjs
```

Pane 배치 파일럿 승인 뒤 동일한 모션 계약을 최종 매뉴얼의 여섯 시나리오에 적용합니다. 캡처는 각 단계 시각과 카메라 초점을 JSON 메타데이터로 남기고, 처리 단계는 원본 동작 속도를 바꾸지 않습니다.

```sh
node scripts/capture-manual-v2-demo-pilot.js
node scripts/process-manual-v2-demo-pilot.mjs
node scripts/check-manual-v2-demo-pilot.mjs
```

첫 번째 명령은 `.tink/current/artifacts/demo-raw/`에 Playwright 원본 WebM을 만듭니다. 두 번째 명령은 프로토타입의 `public/media/demos/`에 아래 결과물을 생성합니다.

- MP4: 1440×900, 30fps, H.264 master
- WebM: 960px 폭, VP9 문서용 영상
- GIF: 800px 폭, 12fps 공유용 대체 파일
- JPG: 각 데모의 완료 상태 포스터
- Contact sheet: `.tink/current/artifacts/demo-processed/`의 6프레임 검토 이미지

## 시나리오

| 이름 | 시작 상태 | 핵심 동작 | 완료 상태 |
| --- | --- | --- | --- |
| workbench | 프로젝트 홈 | Quick open으로 README 선택 | 문서가 Preview로 열림 |
| split | 문서 위·터미널 아래 | 터미널 탭바를 오른쪽으로 드래그 | 문서와 터미널의 좌우 배치 |
| guide-split | 두 문서 탭 | beta.md를 오른쪽 가장자리로 드래그 | 좌우 분할과 독립 탭 표시 |
| terminal | 문서와 닫힌 터미널 | 터미널 재오픈, 새 셸, 명령 실행 | 셸 출력 표시 |
| diff | Git 기준선에서 수정된 문서 | Preview와 Diff 켜기 | 추가 내용과 Changes 레일 표시 |
| search | 열린 문서 | `⌘⇧F`, 본문 검색, 결과 선택 | 검색된 파일로 이동 |

## 카메라와 편집 기준

- 캡처 메타데이터의 실제 시나리오 시작·종료 시각으로 앱 초기화 구간만 제거한다.
- 시나리오별 입력, 조작, 미리보기와 결과 시간을 따로 기록한다. 영상별 최종 길이는 약 4–10초 범위에서 달라진다.
- 확대가 필요한지 먼저 판단하고, 입력 대상이 화면의 약 40%보다 작거나 레이블을 읽어야 할 때만 1.25–1.55배를 사용한다.
- Pane 분할처럼 결과가 공간 관계인 작업은 드롭 전에 전체 화면으로 돌아와 미리보기와 최종 결과를 함께 보여 준다.
- 시작 상태는 300–500ms, 최종 결과는 1.0–1.5초 유지한다.
- 음성, 효과음, 장식 자막은 넣지 않는다.
- 기능 이름과 설명은 매뉴얼 본문에서 제공하고 영상은 동작 자체에 집중한다.

기계 판독 가능한 전체 기준과 파일럿 타임라인은 `demo-motion-spec.json`을 기준으로 합니다.
