What's New

1.0.26 → 1.0.29 변경사항

최근 네 버전에서 문서 검토, Diff 확인, 편집기, 지침 관리 흐름이 어떻게 바뀌었는지 정리했습니다.

DocPilot 1.0.29

현재 버전
  • 01
    AsciiDoc 사용 안내를 보강했습니다

    .adoc, .asciidoc, .asc 파일의 프리뷰와 편집, 코드블록, 긴 문서 작업 방식을 매뉴얼에서 한곳에 정리했습니다.

DocPilot 1.0.28

AsciiDoc · 프리뷰
  • 01
    큰 AsciiDoc 문서가 바로 열립니다

    AsciiDoc 변환을 별도 worker로 옮기고 프리뷰 계산을 줄여, 큰 manual.adoc 파일도 열 때마다 오래 멈추지 않습니다.

  • 02
    프리뷰 스크롤이 더 가벼워졌습니다

    목차 동기화와 라인 표시 작업을 스크롤 흐름에 맞게 조정해 긴 문서를 빠르게 내려도 끊김을 줄였습니다.

  • 03
    편집모드 하이라이트를 보강했습니다

    AsciiDoc, Markdown, JSON, JavaScript, TypeScript 편집 화면에 VSCode 2026 Light/Dark 기반 색상 팔레트를 적용했습니다.

  • 04
    프리뷰와 편집 위치를 이어갑니다

    프리뷰에서 편집으로, 편집에서 프리뷰로 전환할 때 현재 보고 있던 위치에 가깝게 이동합니다.

  • 05
    문서 패널과 코드블록 가독성을 다듬었습니다

    NOTE/WARNING 정보 패널, 코드블록 강조색, 제목 크기 단계를 라이트/다크 테마에 맞춰 다시 조정했습니다.

DocPilot 1.0.27

편집기 · 홈 · 단축키
  • 01
    들여쓰기 선택지를 간단하게 정리했습니다

    상단 Tab 메뉴와 명령 팔레트에서 스페이스 2칸, 스페이스 4칸, 탭 2칸, 탭 4칸을 바로 선택할 수 있습니다.

  • 02
    JSON과 JavaScript 파일을 작업할 수 있습니다

    파일 트리, 빠른 이동, 편집기에서 JSON과 JavaScript 파일을 열고 수정할 수 있습니다. JSON은 보기 모드에서 읽기 좋게 정렬해 보여줍니다.

  • 03
    프리뷰는 Markdown과 JSON에 집중했습니다

    JavaScript, YAML, 일반 텍스트 파일은 기본적으로 편집 모드로 열립니다. 프리뷰가 어울리는 문서와 데이터 파일에만 프리뷰 전환을 제공합니다.

  • 04
    편집기 색과 배경을 다시 맞췄습니다

    다크 테마 편집 배경을 덜 진하게 조정하고, 편집 모드 코드 색상을 프리뷰 코드블록과 더 가깝게 맞췄습니다.

  • 05
    들여쓰기 영역을 눈으로 확인할 수 있습니다

    선행 공백과 탭이 화면에 구분되어 보여, 파일마다 어떤 들여쓰기 기준을 쓰는지 편집 중 바로 확인할 수 있습니다.

  • 06
    파일트리 마지막 항목도 편하게 우클릭할 수 있습니다

    파일트리 하단에 여유 공간을 두고 메뉴 위치를 보정해, 맨 아래 파일에서도 경로 복사 같은 메뉴가 잘리지 않습니다.

  • 07
    홈 이동과 주요 단축키를 정리했습니다

    좌측 상단 DocPilot 로고로 홈 화면에 돌아갈 수 있고, ⌘ + P 빠른 이동과 ⌘ + Shift + P 명령 팔레트 사용법을 매뉴얼에 보강했습니다.

Quick Start

빠른 시작

처음 실행한 뒤 문서 폴더를 열고, 파일을 고치고, 필요한 참고 내용을 Agent에게 넘기는 기본 루프입니다.

Start

🚀첫 작업 흐름

  1. 작업공간 열기에서 로컬 문서 폴더를 선택한다
  2. 파일 탐색과 관리에서 수정할 문서를 연다
  3. 편집과 미리보기에서 내용을 고치고 ⌘ + S / Ctrl + S로 저장한다
  4. 검토가 필요하면 Diff와 변경 비교로 바뀐 부분을 확인한다
  5. Agent에게 넘길 근거는 참고 내용 복사에서 클릭하거나 드래그해 복사한다

핵심 단축키

  • ⌘ + S / Ctrl + S — 현재 파일 저장
  • ⌘ + P / Ctrl + P — 빠른 파일 이동
  • ⌘ + Shift + P / Ctrl + Shift + P — 명령 팔레트 열기
  • ⌘ + F / Ctrl + F — 현재 보기에서 검색

권장 루프

큰 문서를 한 번에 처리하기보다 파일을 열고, 짧게 고치고, 저장하고, Diff로 확인하는 순서로 반복합니다. Agent에게 요청할 때는 필요한 문단만 복사해 파일 경로와 라인 번호가 함께 전달되게 합니다.

Workspace

작업공간 열기

DocPilot은 사용자가 선택한 로컬 폴더를 작업공간 기준 경로로 삼고, 그 안의 문서를 탐색·편집합니다.

Start

📂폴더 열기

시작 화면에서 로컬 문서 폴더를 선택하면 파일 트리와 편집 화면이 해당 폴더 기준으로 열립니다.

  1. 시작 화면에서 폴더 열기를 누른다
  2. 편집할 문서가 있는 로컬 폴더를 선택한다
  3. DocPilot이 해당 경로를 workspace root로 지정하고 bridge 프로세스를 그 경로 기준으로 띄운다
  4. React editor 창이 자동으로 열린다

산출물

📌폴더를 열면 정해지는 것

  • workspace root. 선택한 폴더 경로가 이후 모든 파일 조회의 기준이 된다.
  • 최근 작업공간 기록. 연 폴더가 시작 화면의 최근 폴더 목록에 자동으로 남는다.

탐색

🏠상단 DocPilot 로고로 홈 이동

좌측 상단의 DocPilot 로고를 누르면 열린 파일과 분할 화면을 비우고 홈 상태로 돌아갑니다. 작업 폴더 연결은 유지되므로 파일 트리에서 바로 다시 문서를 고를 수 있습니다.

Boundary: 저장되지 않은 변경사항이 있으면 홈 이동을 막고 안내 메시지를 표시합니다. 먼저 저장하거나 해당 탭을 닫은 뒤 다시 누르세요.

실제 사례

💡다른 작업공간으로 바꾸기

다른 문서 묶음을 작업하려면 홈으로 돌아가 새 폴더를 열면 됩니다. 새 창에서 다른 폴더를 열어도 기존 창의 작업공간은 유지됩니다.

Workspace

파일 탐색과 관리

Workspace Sidebar는 선택한 폴더 하위의 문서를 트리 구조로 보여줍니다. Markdown, JSON, JavaScript, YAML 파일은 확장자에 맞는 아이콘으로 구분됩니다.

Start

🗂️탐색

  • 폴더와 파일을 트리에서 펼치고 클릭해 연다
  • 검색창에 이름을 입력해 파일을 좁혀 찾는다
  • 펼침 / 접기 버튼으로 트리 전체를 한 번에 열고 닫는다
  • 수정된 파일과 닫힌 폴더 안의 수정 상태는 오른쪽 점으로 표시된다

비교

🖱️우클릭 메뉴 — 대상별로 다르다

트리를 우클릭하면 대상(빈 영역/폴더/파일)에 따라 다른 메뉴가 뜹니다.

대상사용 가능한 메뉴
빈 영역파일 만들기, 폴더 만들기, 워크스페이스 추가, Finder에서 열기
폴더파일 만들기, 폴더 만들기, 이름 바꾸기, 폴더 삭제, Finder에서 열기, 경로 복사
파일분할로 열기, 파일 만들기, 이름 바꾸기, 파일 삭제, Finder에서 위치 열기, 경로 복사

권한/제한

🔐접근 경계

Boundary: renderer(React 화면)는 파일시스템에 직접 접근하지 않는다. 모든 조회·생성·삭제 요청은 bridge(Electron 메인 프로세스)를 거친다.

실제 사례

💡새 폴더를 만들고 문서를 정리하기

빈 영역을 우클릭해 폴더 만들기로 하위 폴더를 만들면 예시 문서가 함께 만들어져 트리에 바로 표시됩니다. 그 폴더를 다시 우클릭해 파일 만들기로 문서를 추가합니다. 이름을 입력하지 않고 다른 곳을 클릭하거나 Esc를 누르면 생성 전 상태로 돌아갑니다.

Workspace

최근 폴더와 추가 폴더

자주 여는 문서 폴더를 빠르게 다시 열고, 필요하면 현재 작업공간에 다른 폴더를 추가해 함께 탐색합니다.

최근 폴더

🕒다시 이어서 작업하기

DocPilot은 최근에 열었던 작업 폴더 목록을 시작 화면에 보여줍니다. 같은 문서 묶음을 다시 작업할 때는 폴더 선택 대화상자를 거치지 않고 최근 목록에서 바로 열 수 있습니다.

  • 폴더를 열면 최근 작업공간 목록에 자동으로 기록된다
  • 좌측 상단 DocPilot 로고로 홈에 돌아가도 작업 폴더 연결은 유지된다
  • 저장되지 않은 변경사항이 있으면 홈 이동을 막고 먼저 저장하도록 안내한다

추가 폴더

➕현재 작업공간 확장

파일 트리의 빈 영역을 우클릭한 뒤 워크스페이스 추가를 선택하면 다른 폴더를 함께 탐색할 수 있습니다. 관련 문서가 여러 위치에 나뉘어 있을 때 한 화면에서 파일을 찾는 용도입니다.

Boundary: 추가 폴더도 bridge를 통해 읽고 쓰며, 파일 작업은 사용자가 연 작업공간 범위 안에서만 수행됩니다.

실제 사례

💡문서와 참고 자료를 같이 보기

제품 문서 폴더를 기본 작업공간으로 열고, 별도 참고 자료 폴더를 추가합니다. 파일 트리에서 두 위치를 오가며 문서를 열고, 필요한 참고 문단은 참고 내용 복사로 모아 Agent에게 전달합니다.

Document Work

편집과 미리보기

CodeMirror 6 기반 에디터와 Markdown, AsciiDoc, JSON 프리뷰를 함께 제공합니다. JavaScript, TypeScript, YAML, 일반 텍스트 파일은 편집 중심으로 다룹니다.

Start

📝편집 · 미리보기

  • CodeMirror 6 기반 문법 하이라이트로 Markdown, AsciiDoc, JSON, JavaScript, TypeScript 파일을 편집한다
  • Markdown은 markdown-it 기반 프리뷰가 수정 즉시 렌더링 결과를 반영한다
  • AsciiDoc은 Asciidoctor 기반 프리뷰로 제목, 목록, NOTE/WARNING, 코드블록, 이미지가 문서 형태로 렌더링된다
  • JSON은 프리뷰에서 pretty JSON으로 정렬되어 읽기 쉽게 표시된다
  • JavaScript, TypeScript, YAML, 일반 텍스트는 기본적으로 편집 모드로 열린다

AsciiDoc 지원

📘.adoc 문서 프리뷰

.adoc, .asciidoc, .asc 파일은 편집과 프리뷰를 모두 사용할 수 있습니다. 큰 매뉴얼 문서도 프리뷰 변환을 별도 worker에서 처리해, 파일을 열거나 다시 돌아올 때 앱 전체가 멈추지 않도록 구성되어 있습니다.

항목동작
프리뷰 렌더링Asciidoctor 변환 결과를 문서 화면에 표시
편집 하이라이트제목, 속성 블록, NOTE/WARNING, 리스트, 인라인 코드, 상수형 토큰을 구분
코드블록highlight.js 기반 색상으로 source block을 표시

Tip: 프리뷰와 편집 모드를 오갈 때 현재 보고 있던 위치에 가깝게 이동하므로, 긴 manual.adoc에서도 같은 문맥을 이어서 확인할 수 있습니다.

Diff와 변경 비교

편집 중인 문서의 기준 내용과 현재 내용을 비교하고, 변경 위치를 빠르게 찾습니다.

Diff 보기

🔍바뀐 부분만 확인

Diff를 켜면 기존 내용과 현재 내용을 비교해 추가·삭제·수정된 부분을 표시합니다. Markdown 프리뷰에서는 문단, 표, 목록 같은 블록 단위로 바뀐 부분을 보기 좋게 보여줍니다.

  • 작은 변경은 문서 전체가 아니라 해당 블록 중심으로 표시된다
  • 오른쪽 레일의 변경 표시를 보고 긴 문서 안의 변경 위치를 먼저 파악할 수 있다
  • 편집 모드에서는 raw diff로 저장 전후 텍스트 차이를 확인한다

분할 Diff

↔기존/변경 나란히 보기

분할 Diff를 사용하면 기존 내용과 변경 내용을 좌우로 놓고 비교합니다. 긴 문서나 표처럼 앞뒤 문맥이 중요한 파일에서 유용합니다.

실제 사례

💡저장 전에 검토하기

문서를 수정한 뒤 저장하기 전에 Diff를 켜고 바뀐 부분만 확인합니다. Agent가 바꾼 파일은 변경사항 검토에서 diff 확인, 병합, 수락 또는 거부 흐름으로 처리합니다.

Copy & Send

참고 내용 복사

프리뷰에서 클릭하거나 드래그하면 파일 위치, 라인 번호, 내용이 함께 클립보드에 복사됩니다. 여기서 말하는 Agent는 터미널에서 따로 띄운 Claude Code나 Codex 같은 외부 CLI 세션입니다 — 앱 내장 기능이 아닙니다.

👆클릭 한 번으로 복사

프리뷰에서 문단·리스트·표·코드블록 같은 블록을 클릭하면 즉시 클립보드에 복사되고 우측 하단에 "복사됨" 표시가 뜹니다.

복사되는 형식은 항상 같습니다.

File: guides/architecture.md

(선택한 내용)

비교

🖱️클릭 vs 드래그

블록 전체가 필요하면 클릭, 블록 중 일부만 필요하면 드래그로 범위를 고릅니다.

동작결과
블록 클릭블록 전체를 즉시 복사, "복사됨" 표시
드래그 선택선택한 범위만 팝오버로 확인 후 "복사" 또는 "선택 내용 전체 복사"

산출물

🗂️여러 개 모아서 한 번에 복사

클릭·드래그로 고른 내용은 우측 참고 목록에 chip으로 쌓입니다. 하나씩 복사하지 않고 모아뒀다가 한 번에 복사할 수 있습니다.

  • 복사. 쌓인 항목 전체를 한 번에 클립보드로 복사한다
  • 모두 제거. 참고 목록을 비운다
  • chip의 ×로 항목을 하나씩 뺄 수 있다

권한/제한

Boundary: 활성 지침이 하나라도 있으면, 복사되는 텍스트 맨 앞에 "활성 지침을 먼저 확인하고 반영하라"는 안내와 지침 목록이 자동으로 붙는다. 활성 지침이 없으면 본문만 복사된다. 자세한 지침 설정은 지침과 프리셋 참고.

실제 사례

💡파일 위치를 몰라도 되는 이유

마크다운 문서의 특정 문단을 Agent에게 고쳐달라고 할 때, 예전에는 몇 번째 줄인지 직접 찾아서 알려줘야 했습니다. 지금은 프리뷰에서 그 문단을 클릭만 하면 File 경로와 Lines 번호가 이미 붙은 채로 복사되므로, 터미널의 Agent 세션에 붙여넣기만 하면 됩니다.

지침과 프리셋

Agent에게 항상 전달할 규칙을 등록해두면, 복사할 때마다 자동으로 함께 붙습니다.

Start

➕지침 추가

  1. 왼쪽 지침 탭으로 전환한다
  2. 지침 이름과 본문을 입력하거나, 파일 불러오기로 .md / .txt / .yaml 파일 내용을 채운다
  3. 지침 저장을 누르면 목록에 추가되고 바로 활성(ON) 상태가 된다

비교

⚖️개별 지침 vs 프리셋

단위동작
개별 지침ON/OFF 버튼으로 즉시 켜고 끈다. 켜진 지침만 복사 시 포함된다.
프리셋저장 시점에 켜져 있던 지침들을 프로젝트/전역 범위로 묶어 저장. 적용 버튼으로 그 조합을 한 번에 전환한다.

권한/제한

Boundary: 켜진 지침이 하나도 없으면 프리셋 저장 버튼이 비활성화된다. 프리셋은 항상 "저장 시점에 활성이었던 지침들"만 묶는다 — 나중에 개별 지침을 껐다 켜도 이미 저장된 프리셋 구성은 바뀌지 않는다.

산출물

🔢활성 지침 표시

  • 지침 탭 상단에 N active로 현재 켜진 지침 개수가 표시된다
  • 켜진 지침은 참고 내용 복사할 때마다 텍스트 맨 앞에 이름과 함께 자동으로 나열된다

실제 사례

💡매번 다시 설명하지 않기

"커밋 메시지는 한국어로" 같은 규칙을 지침으로 저장해두면, 이후 프리뷰에서 아무 내용이나 클릭·드래그로 복사할 때마다 이 규칙이 맨 위에 자동으로 붙습니다. Agent에게 매번 다시 설명할 필요가 없습니다.

Review

변경사항 검토

Agent가 만든 변경이나 디스크에서 감지된 변경을 바로 적용하지 않고, 파일별로 diff를 확인한 뒤 병합·거부·수락합니다.

Start

🔍변경 결과 목록

검토할 파일이 생기면 변경 결과 패널에 파일별 카드가 표시됩니다. 각 카드는 변경 전후 줄 수와 증가·감소량을 보여주며, Agent 변경인지 디스크 변경인지 구분합니다.

  • diff 뷰로 확인. 해당 파일을 열고 변경 전 내용을 기준으로 Diff를 켠다
  • 직접 병합. 병합 편집기를 열어 최종 내용을 직접 조정한다
  • 병합 저장. 직접 조정한 내용을 파일에 저장한다
  • 거부. 대기 중인 변경을 적용하지 않고 검토 목록에서 제거한다
  • 수락. 변경된 내용을 그대로 적용한다

권장 루프

✅먼저 보고 결정하기

  1. diff 뷰로 확인으로 변경 범위를 본다
  2. 그대로 적용해도 되면 수락을 누른다
  3. 일부만 반영해야 하면 직접 병합으로 최종 내용을 편집하고 병합 저장을 누른다
  4. 필요 없는 변경이면 거부한다

권한/제한

Boundary: 외부 변경이나 Agent 변경은 자동으로 기존 파일을 덮어쓰지 않습니다. 사용자가 diff 확인, 병합 저장, 거부, 수락 중 하나를 선택해야 최종 상태가 정해집니다.
Safety

안전 모델

DocPilot은 renderer(React UI)와 파일시스템 접근을 분리해 예기치 않은 파일 손상을 막습니다.

권한/제한

🔒권한 경계

Boundary: Renderer는 filesystem/process에 직접 접근할 수 없다. 모든 파일 작업은 Bridge를 거치고, Bridge는 workspace root guard로 범위 밖 접근을 차단한다.
  • renderer — React 화면. 파일시스템/프로세스 권한이 없다.
  • bridge — Electron 메인 프로세스. 파일 조회·생성·삭제를 대신 수행한다.
  • workspace root guard — 열린 폴더 밖의 경로 접근을 막는다.

산출물

📦패키징 범위

  • 패키지에는 React renderer만 포함되며 legacy browser editor는 제외한다

실제 사례

💡외부에서 파일이 바뀌었을 때

편집 중인 문서를 다른 프로그램(예: 텍스트 편집기, git checkout)이 건드리면 DocPilot은 그 변경을 자동으로 덮어쓰지 않습니다. 대신 conflict 리뷰로 표시해, 디스크의 변경 내용을 확인한 뒤 사용자가 직접 반영 여부를 결정하게 합니다.
