# Getting Started

docpilot을 처음 사용하는 분을 위한 빠른 시작 안내입니다.

## 1. 설치 확인

별도의 패키지 설치는 필요 없습니다. Node.js와 Claude Code CLI만 있으면 됩니다.

```bash
node --version      # 18 이상
claude --version    # Claude Code CLI
codex --version     # Codex CLI (선택 사항)
```

## 2. Claude Code 스킬 설치

`/docpilot`으로 바로 열고 싶다면 한 번만 설치합니다.

PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/docpilot.ps1 install-skill
```

macOS/Linux:

```bash
sh scripts/docpilot.sh install-skill
```

Claude Code가 이미 실행 중이면 재시작한 뒤 사용할 수 있습니다.

```
/docpilot /path/to/your/docs
```

## 3. 한 번에 시작

스킬 없이도 아래 명령으로 bridge 시작과 에디터 열기를 한 번에 처리할 수 있습니다.

PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/docpilot.ps1 open /path/to/your/docs
```

macOS/Linux:

```bash
sh scripts/docpilot.sh open /path/to/your/docs
```

인자 없이 실행하면 현재 디렉터리 하위에서 `.md` 파일이 있는 폴더를 찾아 선택하게 해줍니다.

## 4. bridge 직접 시작

편집할 마크다운 파일이 있는 디렉터리를 `--root`로 지정해 서버를 실행합니다.

```bash
node bridge.js --root /path/to/your/docs
```

터미널에 아래와 같이 출력되면 준비된 것입니다.

```
docpilot bridge  http://localhost:7474
root             /path/to/your/docs
```

## 5. 에디터 직접 열기

```bash
open editor.html
```

헤더 좌측 상태 표시등이 **bridge on** (초록)으로 바뀌면 연결이 완료됩니다.

## 6. 기본 편집 흐름

### 파일 열기

왼쪽 파일 트리에서 `.md` 파일을 클릭합니다. 폴더 옆 화살표를 클릭하면 하위 폴더를 펼칩니다.

파일이 열리면:
- 중앙 프리뷰에 렌더링된 마크다운이 표시됩니다.
- 좌측 목차 패널에 헤딩 구조가 나타납니다.

### 텍스트 선택 → 칩 만들기

수정하고 싶은 부분을 선택해 칩으로 만드는 방법은 세 가지입니다.

| 방법 | 동작 |
|------|------|
| 텍스트 드래그 | 마우스로 드래그하면 선택된 텍스트가 칩으로 추가됩니다 |
| 문단 클릭 | 문단·제목·목록 항목을 클릭해도 칩이 생성됩니다 |
| 목차 더블클릭 | 목차 항목을 더블클릭하면 해당 섹션 전체가 칩으로 추가됩니다 |

선택 칩은 프리뷰 우측 상단에 떠 있습니다.

### 드로우 모드

손으로 영역을 직접 그려서 여러 블록을 한번에 선택하고 싶을 때 사용합니다.

1. 툴바의 **✦ 드로우** 버튼을 클릭하거나 **Cmd+D** 를 누릅니다.
2. 화면에 레이저 포인터가 나타납니다.
3. 선택하고 싶은 텍스트 영역을 드래그로 그립니다.
4. 마우스 버튼을 놓으면 선이 지나친 텍스트 블록이 칩으로 추가됩니다.

> Cmd+D를 누르는 동안만 드로우 모드를 임시로 사용할 수도 있습니다.

### 지시하기

1. 칩을 클릭해 활성화합니다 (파란 테두리).
2. **✦ 지시하기** 버튼을 누릅니다.
3. 수정 지시를 입력하고 **보내기**를 누릅니다.

Codex CLI가 설치되어 있으면 Claude와 Codex가 동시에 수정안을 생성합니다.

### 지침 등록

왼쪽 **지침** 패널에서 앞으로 모든 지시 결과가 따라야 할 규칙을 등록할 수 있습니다. 지침은 문서 루트의 `.docpilot/instructions.json`에 저장됩니다.

| 방법 | 동작 |
|------|------|
| 직접 입력 | 제목과 본문을 작성해 지침으로 저장합니다 |
| AI 정리 | 자연어 요구를 짧고 명확한 지침으로 정리한 뒤 검토하고 저장합니다 |
| 파일에서 가져오기 | 마크다운 파일을 선택해 지침 본문으로 가져옵니다 |
| ON/OFF | 지침을 삭제하지 않고 적용 여부만 전환합니다 |

### 수정안 검토

오른쪽 **작업 히스토리** 패널에서 결과를 확인합니다.

- **미리보기** — 수정된 전체 문서를 별도 창에서 미리 봅니다.
- **적용** — 수정안을 현재 파일에 반영합니다.
- **거부** — 수정안을 버립니다.
- **조합** (Claude+Codex 모두 완료 시) — 두 결과를 하나의 최종안으로 병합합니다.

## 7. 패널 조절

작업 환경에 맞게 패널 크기를 조절할 수 있습니다.

- 패널 헤더의 **‹ / ›** 버튼으로 패널을 접고 펼칩니다.
- 패널 경계를 드래그해 폭을 자유롭게 조절합니다.

## 8. 편집 모드

**편집** 버튼을 누르면 원문 textarea와 라이브 프리뷰를 나란히 볼 수 있습니다.

직접 수정 후 **저장** 버튼으로 파일에 씁니다. 저장 후 자동으로 보기 모드로 돌아갑니다.

## 9. 칩 관리

| 동작 | 설명 |
|------|------|
| 클릭 | 활성화/비활성화 토글 |
| 압정 아이콘 클릭 | 고정 — 지시 후에도 칩이 남아 있습니다 |
| ✕ 클릭 | 즉시 삭제 |
| 전체 선택 | 현재 문서 전체를 칩으로 추가 |

비활성 칩은 지시에 포함되지 않습니다. 고정되지 않은 활성 칩은 지시 후 자동 삭제됩니다.
